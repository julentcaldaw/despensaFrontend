# AGENTS Operating Guide

## Scope
Agents support implementation, documentation, refactoring, and review tasks in this repository.

## Source of truth
- Product context: `docs/product/`
- Data rules: `docs/data/`
- Frontend architecture: `docs/frontend/`
- UI conventions: `docs/ui/`
- Coding conventions: `docs/conventions/`
- Delivery process: `docs/agents/`

## Implementation baseline
- Frontend stack: React + TypeScript.
- UI stack: Tailwind CSS v4 + daisyUI.
- User-facing copy in the app must be written in Spanish.

## Priorities
1. Correctness and working behavior.
2. Minimal, focused changes.
3. Consistency with documented architecture.
4. Explicit handling of errors and role-based logic.

## Permissions and constraints
- Do not perform destructive repository actions unless explicitly requested.
- Do not change unrelated files.
- Keep docs aligned with behavioral changes.

## Expected outputs
- Working code changes.
- Updated documentation when relevant.
- Clear summary of decisions and tradeoffs.
