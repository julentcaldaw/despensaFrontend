# UI Principles

## Design direction
Warm and approachable minimal utility. Prioritize readability, task speed, and emotional clarity.

## Approved design decisions (v1)
- Brand personality: warm and close.
- Visual style: minimal utilitarian.
- Color direction: terracotta + cream neutrals.
- Typography tone: modern, highly legible sans-serif.
- Density: comfortable spacing by default.
- Motion level: moderate, always functional.
- Empty/error tone: close and motivating, still actionable.

## Core principles
- Make core actions obvious on every primary screen.
- Reduce cognitive load with progressive disclosure.
- Keep interactions predictable across modules.
- Prefer clarity over decorative complexity.
- Preserve a warm tone through microcopy and visual rhythm.

## Accessibility baseline
- Meet WCAG 2.1 AA contrast requirements.
- Full keyboard navigability for core flows.
- Clear focus states for interactive elements.
- Semantic HTML and ARIA where needed.

## Interaction rules
- Confirm destructive actions.
- Show loading and empty states intentionally.
- Provide immediate feedback for successful actions.
- Keep error messaging specific and actionable.
- Use short toasts for successful pantry, recipe, and shopping-list updates.
