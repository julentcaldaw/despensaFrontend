# ADR Process

## Purpose
Architecture Decision Records (ADRs) capture significant technical decisions and their consequences.

## When to create an ADR
- Choosing a core library or major pattern.
- Introducing or changing architectural boundaries.
- Making tradeoffs with long-term impact.

## Workflow
1. Copy `0001-template.md` into a new numbered file.
2. Fill context, decision, alternatives, and consequences.
3. Review in pull request with affected stakeholders.
4. Mark final status (`accepted`, `superseded`, `rejected`).

## File naming
`NNNN-short-title.md` (for example: `0002-state-management-strategy.md`).
