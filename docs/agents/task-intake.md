# Task Intake

## Required inputs
- Problem statement.
- Scope boundaries (in/out).
- Affected routes or modules.
- Expected behavior.
- Constraints (deadline, dependencies, forbidden changes).

## Optional but recommended
- Screenshots or UX references.
- API examples.
- Known edge cases.

## Acceptance criteria format
- Use concrete, testable statements.
- Include success and failure cases.
- Identify role-specific behavior where applicable.

## Definition of done reference
Every task must satisfy `docs/agents/definition-of-done.md` before closure.
